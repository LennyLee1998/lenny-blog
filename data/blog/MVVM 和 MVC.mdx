### MVC 是什么

MVC（Model-View-Controller）是一种软件架构模式，广泛应用于开发用户界面系统，尤其适用于Web应用程序或桌面应用程序。MVC 模式将应用程序分为三个主要组件：Model、View 和 Controller。

- **Model**: 负责业务逻辑和数据管理
- **View**: 用户界面，负责展示数据
- **Controller**: 接收用户输入，处理业务逻辑并更新模型和视图。

### MVVM 是什么

MVVM（Model-View-ViewModel）是一种软件架构模式，主要用于构建用户界面，特别是在现代前端开发中，如使用框架的应用程序，比如 Vue.js 和 Angular。

- **Model**: 同样负责数据和业务逻辑。
- **View**: UI层，通常与用户界面元素绑定。
- **ViewModel**: 介于Model和View之间，负责处理与View的交互逻辑，通常通过**数据绑定**（Data Binding）来实现与View的同步。

![1733656021945](MVVM 和 MVC.assets/1733656021945.png)

### 两者的区别是什么

vue 是典型的 `MVVM ` 架构,vue 在架构中起到的作用就是`ViewModel `, 在使用 vue 的过程中, vue 其实是相当于黑盒一般的存在, View与ViewModel之间通过公共接口进行交互。所以三者是解耦的, 也更加有利于单元测试

而在 `MVC` 中, Controller 直接负责处理来自 View 的用户输入、执行业务逻辑并更新 Model 和 View。每一次用户的操作通常都需要通过 Controller 来手动触发，并依赖Controller 自身的实现。Controller 与 View 和 Model 之间的紧密耦合意味着其内部逻辑更复杂，难以进行单元测试。
